# ✦ Tamara's Portfolio V2

A personal portfolio website built to showcase my growth as a frontend engineer — featuring my projects, skills, and personality.

🔗 **Live Site:** [https://tamara-portfolio-v2.vercel.app/](https://tamara-portfolio-v2.vercel.app/)

---

## ✨ Features

- Responsive design — mobile-first with a hamburger menu for smaller screens
- Smooth scroll navigation with a sticky navbar that reacts to scroll
- Animated gradient blobs in the hero section
- Morphing avatar shape animation
- Infinite marquee strip showcasing my tech stack
- Playful bento grid for skills
- Project cards with colored hover effects and box shadows
- Contact section with social links
- Custom favicon and Google Fonts integration

---

## 🛠️ Technologies Used

- **React.js** — component-based UI
- **Vite** — fast development build tool
- **Tailwind CSS** — utility-first styling
- **CSS Keyframe Animations** — custom animations via Tailwind config
- **JavaScript (ES6+)** — logic and interactivity

---

## 🚀 Getting Started

To run this project locally:

```bash
# Clone the repository
git clone https://github.com/Tamara-tgod/portfolio-v2.git

# Navigate into the project
cd portfolio-v2

# Install dependencies
npm install

# Start the development server
npm run dev
```

Then open [http://localhost:5173](http://localhost:5173) in your browser.

## 🧠 Learning Outcomes

Building this project helped me solidify:

- **React component thinking** — breaking a UI into reusable, isolated pieces
- **Props and data-driven rendering** — using arrays and `.map()` to render dynamic content
- **useState and useEffect** — managing UI state and side effects (scroll detection, menu toggle)
- **Tailwind CSS** — utility classes, responsive prefixes, and extending the config with custom animations, fonts, colors and shadows
- **Responsive design** — mobile-first layouts with `md:` and `lg:` breakpoints
- **Vite** — project scaffolding, asset imports, and production builds
- **Debugging** — resolving dependency conflicts, fixing broken animations, and tracing deployment issues
